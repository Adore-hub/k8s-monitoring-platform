# Architecture

## 架构概览

本项目采用分层监控架构，整体分为以下几层：

### 1. 数据采集层
- Node Exporter：采集节点资源指标
- cAdvisor：采集容器资源指标
- kube-state-metrics：采集 Kubernetes 对象状态指标
- 应用自定义指标：采集业务 HTTP 指标

### 2. 指标存储与计算层
- Prometheus：负责指标抓取、存储、规则计算

### 3. 告警层
- Alertmanager：负责告警路由、分组、抑制与通知

### 4. 可视化层
- Grafana：负责展示集群、节点、Pod 与业务指标

## 设计原则

- 配置与规则解耦
- 监控体系可版本化
- 支持一键部署与快速回滚
- 支持基础设施与业务统一观测

## 高可用能力

项目适配 3 节点 Kubernetes 集群，可在节点故障场景下保持监控服务的持续可用。
